# موجز المنتج: فهيم (Faheem)

## 1) تعريف المنتج
**فهيم** ليس مجرد سجل لقراءات السكر، وليس مجرد حاسبة كارب مستقلة.  
فهيم يربط بين: **القراءات، الأكل، الكارب، الوقت، التكرار، وأنماط المستخدم**؛ ليحوّل البيانات اليومية المتفرقة إلى فهم واضح ومفيد يساعد المستخدم على قراءة يومه بشكل أعمق وبسيط.

**الوعد الأساسي:**  
**فهيم لا يحسب لك فقط… فهيم يفهم يومك معك.**

---

## 2) الجمهور المستهدف

**الجمهور الأساسي (Primary):**
- أمهات وآباء الأطفال المصابين بالسكري النوع الأول في السعودية والخليج.

**الجمهور الثانوي (Secondary):**
- المصابون بالسكري النوع الأول.
- المرافقون.
- المدرسة أو من يعتني بالطفل.
- المهتمون بحساب الكارب وفهم قراءات السكر.

---

## 3) حدود السلامة والمسؤولية الطبية
فهيم منتج توعوي/تنظيمي وليس أداة تشخيص أو علاج.

- فهيم **ليس طبيبًا**.
- فهيم **لا يشخّص** أي حالة صحية.
- فهيم **لا يوصي بجرعات إنسولين**.
- فهيم **لا يستبدل الاستشارة الطبية** أو متابعة الفريق الصحي.

يجب إظهار هذه الحدود بوضوح داخل المنتج، خصوصًا في الصفحات ذات العلاقة بالقراءات والكارب.

---

## 4) الصفحات الظاهرة في MVP

### 4.1 نبض يومك
صفحة يومية دافئة وبسيطة لإدخال قراءة السكر وقراءة السياق.

**العناصر الأساسية:**
- إدخال قراءة السكر.
- اختيار/إدخال سياق القراءة (مثل: قبل الأكل، بعد الأكل، نشاط، صيام...).
- عرض الأثر المرتبط بالسياق.
- متوسط القراءة.
- الاتجاه (Trend).
- نبض اليوم (ملخص بصري سريع لحالة اليوم).
- ملاحظة ذكية (Smart Note) مبنية على الأنماط المحلية.
- سجل مطوي (Folded Log) يمكن توسيعه لمراجعة اليوم/الأيام السابقة.

### 4.2 كم كارب؟
صفحة إنقاذ سريعة لحساب الكارب أثناء الأكل أو قبله.

**العناصر الأساسية:**
- حقل البحث الرئيسي: **«وش الأكل؟»**.
- اقتراحات فورية أثناء الكتابة.
- إدخال الكمية.
- نتيجة الكارب لكل عنصر.
- سلة الوجبة (Meal Basket).
- مجموع الكارب الكلي للوجبة.
- بطاقة مشاركة (Share Card).

### 4.3 فهيم بلس
متاح ومفتوح في النسخة الأولى **بدون أي لغة دفع**.

**المحتوى الأساسي:**
- قسم **«اسأل فهيم»**.
- إجابات **Rule-based** ناتجة من البيانات المحلية المخزنة داخل الجهاز.
- لا يوجد API ذكاء اصطناعي خارجي في MVP.

**Prompts جاهزة داخل “اسأل فهيم”:**
- حلّل يومي
- ليش ارتفعت قراءتي؟
- وش أكثر وجبة أثّرت علي؟
- ساعدني أفهم معامل الكارب
- اشرح لي قراءة بعد الأكل
- وش أراقب اليوم؟

## 5) الطبقات المخفية (البنية الداخلية)

### 5.1 مكتبة فهيم للكارب
محرك معرفة داخلي مخفي للأطعمة والكارب، **وليس ملف JSON بسيط**.

**متطلبات البيانات في المكتبة:**
- أسماء عربية متعددة.
- مرادفات/أسماء بديلة (Aliases).
- علامات تجارية (Brands).
- أطعمة سعودية/خليجية شائعة.
- منتجات غذائية.
- وحدات متنوعة (حبة، كوب، ملعقة، غرام...).
- قيمة الكارب لكل 100 غرام.
- درجة ثقة (Confidence).
- ملاحظات داخلية.
- كلمات مفتاحية للبحث.

**سلوك البحث المطلوب في المكتبة:**
- دعم المرادفات (Aliases) بشكل مباشر.
- دعم اللهجات العربية الشائعة.
- دعم اختلافات الكتابة والإملاء المتداولة.
- دعم أسماء العلامات التجارية.
- دعم الأطعمة السعودية/الخليجية المتكررة.
- مطابقة سريعة بأسلوب fuzzy matching قدر الإمكان.

**أمثلة بحث مرجعية:**
- فطيرة، فطاير، لوزين
- رز، عيش، رز أبيض، رز مطبوخ
- كبسة، مندي، بخاري
- بروست، دجاج مقلي
- خبز، تميس، صامولي

> ملاحظة تنفيذية: ملف المكتبة الفعلي سيتم رفعه لاحقًا بعد إعداد **LIBRARY_ARCHITECTURE.md**.

### 5.2 ذاكرة فهيم
طبقة ذكاء محلية مخفية تعتمد على **localStorage** فقط.

**ما يجب تخزينه:**
- القراءات.
- سياقات القراءات.
- عمليات البحث عن الطعام.
- العناصر المختارة.
- الكميات.
- مجموع الكارب.
- الوجبات المتكررة.
- أيام الأسبوع المرتبطة بالأنماط.
- العلاقة بين الوجبات والقراءات اللاحقة.
- المتوسطات.
- فترات الارتفاع.
- فترات الاستقرار.
- الملاحظات الذكية.

### 5.3 نظام أكواد التفعيل
- نظام مستقبلي لتفعيل يدوي شهري عبر كود.
- **لا توجد واجهة دفع** في v1.
- يكتفى في MVP بتهيئة البنية المنطقية/الحدث التحليلي المرتبط بإدخال الكود.

## 7) SEO (التهيئة الحالية والمستقبلية)

**الأساس المعتمد (Foundation):**
- **Title:**
  - فهيم | مساعد يومي لفهم السكر والكارب
- **Meta Description:**
  - فهيم تطبيق ويب عربي يساعد أسر أطفال السكري على فهم قراءات السكر، حساب كارب الوجبات، وملاحظة الأنماط اليومية بطريقة بسيطة ومطمئنة.
- **Keywords:**
  - حساب الكارب، كارب الوجبات، سكري النوع الأول، قراءات السكر، سكر الأطفال، حاسبة الكارب، معامل الكارب، تنظيم يوم السكري، فهم قراءات السكر، حساب كارب الأكل.

**في MVP:**
- بنية عربية RTL صحيحة.
- استخدام عناوين دلالية (Semantic Headings) متسلسلة.

**لاحقًا:**
- Open Graph.
- `sitemap.xml`.
- `robots.txt`.

## 8) التحليلات (لاحقًا)
لا يتم تضمين Google Analytics الآن.

يتم فقط تجهيز نقاط الأحداث (Event Hooks) للاستخدام لاحقًا:
- `reading_added`
- `reading_context_selected`
- `food_search`
- `food_selected`
- `food_quantity_entered`
- `carb_total_calculated`
- `meal_item_added`
- `plus_opened`
- `ask_faheem_used`
- `share_card_clicked`
- `activation_code_entered`

## 9) نطاق MVP (حدود التنفيذ)
النسخة الأولى يجب أن تكون:
- تطبيق ويب RTL ثابت وخفيف (Static Lightweight Web App).
- تخزين محلي فقط عبر `localStorage`.
- بدون Backend.
- بدون تسجيل دخول.
- بدون دفع.
- بدون API ذكاء اصطناعي حقيقي.
- بدون تعرّف صور.
- بدون محرك قرارات طبية.

## 10) خارطة الطريق التنفيذية
الترتيب المقترح للتنفيذ بعد هذا الموجز:
1. إعداد **LIBRARY_ARCHITECTURE.md**.
2. بناء هيكل المشروع (Project Structure).
3. تصميم القشرة البصرية الأساسية (Visual Shell).
4. تنفيذ صفحة **نبض يومك**.
5. تنفيذ صفحة **كم كارب؟**.
6. إضافة مجموعة بيانات ابتدائية (Starter Dataset).
7. تنفيذ طبقة **ذاكرة فهيم**.
8. تنفيذ **فهيم بلس** (Rule-based محلي).
9. استكمال SEO الأساسي.
10. إضافة Hooks التحليلات.
11. النشر (Deployment).

---

## 11) مبدأ التجربة
فلسفة فهيم في MVP: **الوضوح، السرعة، والطمأنينة**.  
كل شاشة يجب أن تساعد المستخدم على فهم يومه بقرار بسيط، وبدون تعقيد تقني، مع احترام كامل لحدود السلامة الطبية.
